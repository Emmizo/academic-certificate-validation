What happens on submit

Certificate data is converted into a canonical string format.
The canonical string is hashed with SHA-256 to create a tamper-evident fingerprint.
The hash is signed using the active RSA-2048 private key, producing a digital signature.
No active RSA key pair is currently available. Please go to the Keys page and generate a key pair before issuing certificates.

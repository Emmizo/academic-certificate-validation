How CertSign works

A simple three‑step flow built on strong cryptography.

1 Issue
Register the award
Admins capture student identity, program, institution, and issue date in the secure console.
2 Sign
Hash & sign the record
CertSign hashes the canonical certificate data with SHA‑256 and signs the hash using RSA‑2048.
3 Verify
Public verification
Anyone can verify the certificate ID on the public portal; any change to the data breaks the signature.

Why institutions use CertSign

Reduce paper, prevent forgery, and give graduates verifiable credentials they can share anywhere.

Fraud resistance

Each certificate is bound to a digital signature; forged PDFs or screenshots fail verification immediately.

Operational simplicity

Single Spring Boot application with an admin console and public verification — no separate frontend to deploy.

Audit‑ready logs

Every verification is logged with timestamp, source IP, and result, helping you answer regulator and auditor queries.
